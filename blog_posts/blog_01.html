Variables

Variables are to Ruby like freckles are to a ginger kid in the sun. Abundant. Luckily it’s a very simple concept to understand.

They are nothing more than containers of data and code. They are labels to be used elsewhere in your code so that you don’t have to repeat it.

result = 10 + 10

The variable here is ‘result’. When you then go on to use the variable elsewhere in your code it will return the value of 20.

second_result = result + 10

The above would return 30, which can then be called using the ‘second_result’ variable.

The convention for naming variables is called lower_snake_case, all lowercase with underscores in lieu of spaces.

Deviate away from convention under the pain of death. You have been warned.
