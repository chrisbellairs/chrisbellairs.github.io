Methods

Keep repeating the same piece of code over and over again? You need methods. Keep your code DRY. Don’t repeat yourself. Not only does it make your code cleaner, but it will also help you improve and add to your code as it grows.

A method is defined as follows…

def method_name
 #your code goes here
end

Just the same as variables your method names should follow lower_snake_case convention.

Once you have defined a method, to call it all you need do is refer back to it. Here’s an example…

def greeting
 puts “Hello world!”
end

greeting

#This would then return the string “Hello world!” to your terminal.

It’s pretty clear how much cleaner and easier to use your code will be by using methods, but there’s one more thing that about methods that increases their usability exponentially.

You can also feed data into the method which will use it in a block of code and spit out the results.

def sum_of(first_num, second_num)
 return first_num + second_num
end

sum_of(5, 10)

#This would return 15.

When defining the method you must provide parameters, in the above this would be (first_num, second_num). Upon calling the method you provide arguments to fill those parameters. Five and ten are fed into the method and the block of code adds them together.

The keyword return indicates what should be sent back. If there is no return given then the method will automatically return the last line of the method.

The examples I have provided are pretty basic, but I’m sure you can see how quickly they can scale to create complex applications.
